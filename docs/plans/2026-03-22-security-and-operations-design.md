## Обзор

DSBot — персональный AI-ассистент Данияра на базе Claude Code + Telegram Plugin. Бот имеет полный доступ к файлам, Bash, Bitrix24, Docker, SSH. Единственная защита — allowlist по Telegram user_id. Инструкция "действуй, не спрашивай" усиливает каждую угрозу.

Два связанных вопроса: (1) как защитить бота от деструктивных действий, (2) где ещё применять DSBot для контроля операционки Winch Group.

## Часть 1: Безопасность

### 1.1 Матрица угроз

| Угроза | Вероятность | Импакт | Текущая защита | Дыра |
|--------|------------|--------|---------------|------|
| Prompt injection (группа/forwarded msg) | HIGH | CRITICAL | Группы не включены | Нет фильтрации контента, "действуй не спрашивай" |
| Украденный телефон / session hijack | MEDIUM | CRITICAL | Allowlist по user_id | Нет второго фактора, нет rate limiting |
| Утечка bot token | MEDIUM | HIGH | assertSendable() | Токен в .env в git-tracked репо |
| Indirect injection через фото/файлы | MEDIUM | HIGH | Нет | CLAUDE.md: "фото + инструкция = делай сразу" |
| Эксфильтрация данных через reply | LOW-MEDIUM | CRITICAL | assertSendable (только files) | Claude может Read любой файл и вставить текст в reply |
| Деструктивные действия (rm, force push) | LOW | CRITICAL | Нет | Нет blocklist, нет подтверждения |
| VPS lateral movement | LOW | CRITICAL | Нет | root, docker socket, SSH ключи |

### 1.2 Решение: трёхуровневая защита

#### Уровень 1: CLAUDE.md — "Красная зона" (делать СЕЙЧАС)

Добавить в CLAUDE.md после "Стиль ответов":

```markdown
## Уровни доступа (обязательно)

Правило "Действуй, не спрашивай" НЕ распространяется на деструктивные операции.

### Tier 1 — автоматически (без подтверждения)
- Read/Grep/Glob файлов Second Brain
- Edit planner/ (tasks, memories, context, backlog)
- Bitrix24 чтение (list, get)
- Telegram reply/react/edit знакомым контактам
- Транскрипция аудио
- Веб-поиск

### Tier 2 — сообщи что делаешь перед выполнением
- Bitrix24 запись (create/update deal, lead, contact)
- Генерация контрактов PDF
- Отправка отчётов (--send)
- Google Calendar create
- git commit / git push
- Edit файлов за пределами planner/

### Tier 3 — СТОП. Опиши действие, жди "да" / "давай"
- Удаление файлов/папок (rm, rmdir, unlink)
- Массовое редактирование (>3 файлов, truncate, очистка)
- Docker деструктив (rm, stop, down, prune)
- Git деструктив (push --force, reset --hard, branch -D)
- SSH деструктив (rm/stop/kill на VPS)
- БД деструктив (DROP, DELETE без WHERE, TRUNCATE)
- Отправка незнакомым контактам
- Изменение credentials (.env, access.json, токены)
- Перезапуск сервисов (systemctl, kill, tmux kill)
- Чтение .env файлов (потенциальная утечка ключей)

### Формат подтверждения Tier 3
"⚠️ Собираюсь: [что именно]. Это необратимо. Подтверди?"
```

**Надёжность:** ~99% для прямых команд, ~90% для ambiguous, ~80% против prompt injection. Достаточно для single-user бота.

#### Уровень 2: Инфраструктура (делать СЛЕДУЮЩАЯ СЕССИЯ)

| # | Действие | Что решает | Усилия |
|---|----------|-----------|--------|
| 1 | `.env` и `.mcp.json` → .gitignore | Утечка секретов через git | 5 мин |
| 2 | Pre-sync auto-commit hook | Потеря данных при blind sync | 30 мин |
| 3 | Создать user `dsbot`, убрать root | Привилегии | 2 часа |
| 4 | Restricted docker exec wrapper | Container escape | 1 час |

#### Уровень 3: Bitrix read-only webhook (делать ПОЗЖЕ)

Текущий webhook в `.mcp.json` имеет полный REST доступ. Создать отдельный webhook с read-only scopes (`crm.deal.list`, `crm.lead.list`, `crm.contact.list`) для DSBot. Write-операции — через отдельный webhook, доступный только по Tier 2.

## Часть 2: Операционное применение

### 2.1 Производство (эксперт: Голдратт)

**Bottleneck:** скорость обратной связи = 12-24 часа (отчёт вечером → дашборд утром → реакция завтра).

**Архитектура: гибрид winch-bot (push) + DSBot (pull)**

| Use Case | Кто | Бот | Приоритет | Усилия |
|----------|-----|-----|-----------|--------|
| **UC1: Андон** — алерт при KPI >200% или <30% сразу после отчёта | Оксана | winch-bot | P1 | 1 день |
| **UC2: Production brief** — секция в утреннем брифинге | Данияр | DSBot | P1 | 3 часа |
| **UC3: Молчуны** — кто не отчитался к 19:00 | Оксана | winch-bot | P2 | 4 часа |
| **UC4: Constraint report** — ТОП-3 узких места за неделю | Данияр | winch-bot | P2 | 4 часа |
| **UC5: Дедлайн заказов** — Bitrix дедлайн через <=2 дня | Данияр+Оксана | DSBot | P2 | 5 часов |
| **UC6: "Зарплата Зилола"** — быстрый запрос KPI | Данияр | DSBot | P3 | 2 часа |
| **UC7: Коррекция норм** — предложение обновить нормы по 4-недельной статистике | Данияр | winch-bot | P3 | 1 день |

**Почему гибрид:** winch-bot уже имеет Telethon (подключён к чату отчётов), aiogram, scheduler. DSBot имеет Bitrix MCP и Claude Code. Каждый делает то, для чего лучше приспособлен.

### 2.2 Продажи (эксперт: Nir Eyal)

**Bottleneck:** конверсия лид → сделка → оплата. Менеджеры — ограничение системы.

| Use Case | Кто | Как | Приоритет |
|----------|-----|-----|-----------|
| **Stale deal alerts** (усилить) — добавить имя менеджера, последнюю активность | Данияр | proactive_alerts.py | P1 |
| **Win/loss analysis** — 5 won ($X), 3 lost (причины) | Данияр | CLAUDE.md + Bitrix MCP `generate_sales_report` | P2 |
| **Pipeline velocity** — bottleneck по стадиям | Данияр | Bitrix MCP `track_deal_progression` | P2 |
| **Быстрое КП** — "КП для Artel: пакеты 1000шт крафт" | Менеджер | contract/generator.py + Newcalc | P3 |
| **Auto-follow-up** — КП без ответа 48ч | Менеджер | Bitrix activities API | P3 |

**Решение:** Расширить DSBot (вариант A). Bitrix MCP уже имеет 50+ tools. SalesBot продолжает работать с клиентами (B2C), DSBot — контроль (B2B internal).

### 2.3 Управление командой (эксперт: Далио)

**Bottleneck:** внимание Данияра. 59 задач, 8 делегированных без трекинга.

**Фазовый план:**

#### Фаза 1: Proactive-only (0 риска, 1-2 дня)
| Use Case | Что | Как |
|----------|-----|-----|
| **Эскалация-пинг** | Задача `[d]` без обновления >3 дней → пинг исполнителю, >5 дней → эскалация Данияру | proactive_alerts.py |
| **Персональные weekly** | Оксане — производственный блок, Лайло — административный | send_briefing.py |
| **Decision log** | "Запиши решение" → Knowledge/decisions/ + уведомление затронутым | CLAUDE.md |

#### Фаза 2: Self-service для Оксаны (3-5 дней)
- Добавить Оксану в allowFrom
- CLAUDE.md: если user != Данияр → только ReportsAnalyze + нормы операций
- Тестировать 1 неделю, собирать логи

#### Фаза 3: Отдельный бот (по результатам)
- Если prompt-based ограничения протекают → @dsbot_ops с отдельным CLAUDE.md
- Только производство, нормы, KPI

## Метрики успеха

### Безопасность
- 0 инцидентов деструктивных действий через Telegram
- 0 утечек секретов через git
- Время восстановления при случайном удалении: <5 мин (git rollback)

### Производство
- Время обнаружения аномальной выработки: 12-24ч → 5 мин (андон)
- % сотрудников отчитавшихся вовремя: трекинг → рост до 95%
- Кривые нормы: исправляются за дни, не за недели

### Управление
- Делегированные задачи без обновления >5 дней: 0
- Запросы Оксаны к Данияру за данными: -50% (self-service)
- Время Данияра на ручной контроль: -30 мин/день
