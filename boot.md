# DSBot Boot Protocol

Выполни все фазы по порядку. Каждая фаза ДОЛЖНА завершиться перед следующей.
Ошибки в фазах НЕ останавливают бут — логируй и продолжай.

## Phase 1: Ориентация

1. Выполни `date '+%Y-%m-%d %H:%M:%S %Z'` — запомни текущую дату/время
2. Часовой пояс: Asia/Tashkent (UTC+5)
3. Прочитай CLAUDE.md — загрузи все инструкции, правила, tier-доступ

## Phase 2: Восстановление памяти

1. Read `planner/context.md` — что обсуждали в прошлой сессии?
2. Read `planner/tasks.md` — посчитай просроченные задачи (срок < сегодня)
3. Read `planner/memories.md` — обнови персистентные факты
4. Запомни: контекст восстановлен, ты в курсе всех текущих дел

## Phase 3: Восстановление cron-задач

1. Выполни `crontab -l` — проверь что cron-задачи на месте
2. Ожидаемые cron-записи:
   - `0 2 * * *` → send_briefing.py morning (07:00 Tashkent)
   - `0 7 * * *` → send_briefing.py midday (12:00 Tashkent)
   - `0 16 * * *` → send_briefing.py evening (21:00 Tashkent)
   - `0 15 * * 0` → send_briefing.py weekly (Вс 20:00 Tashkent)
   - `*/5 * * * *` → limits_tracker.sh
   - `*/15 * * * *` → server_monitor.sh
3. Если чего-то не хватает — используй CronCreate для восстановления

## Phase 4: Проверка фоновых процессов

1. Выполни `docker ps --format '{{.Names}} {{.Status}}'` — какие контейнеры работают
2. Проверь ключевые сервисы:
   - `winch-bot` — ReportsAnalyze (должен быть Up)
   - `salesbot` — SalesBot (должен быть Up)
   - `parsing-seo` — парсер (должен быть Up)
3. Если что-то down — залогируй, НЕ перезапускай без подтверждения (Tier 3)

## Phase 5: Утренний брифинг

1. Проверь текущее время
2. Если 05:00–09:00 Tashkent:
   - Покажи просроченные P1 задачи
   - Покажи задачи на сегодня
   - Если есть Bitrix MCP — покажи активные сделки
3. Если вне этого окна — пропусти с логом "Не утро, брифинг пропущен"

## Phase 6: Сигнал "Online"

Отправь в Telegram сообщение:

```
DSBot online ✓

Время: [дата и время]
Контекст: [тема из context.md или "чистый старт"]
Просроченных задач: [число]
Docker: [список работающих контейнеров]
Cron: [OK / N missing]
```

## Phase 7: Обновление состояния

1. Обнови `planner/context.md`:
   - Последнее обновление: [текущая дата и время]
   - Тема: Boot — DSBot перезапущен
   - О чём говорили: Автоматический запуск, все фазы пройдены
2. Лог: "Boot protocol completed successfully"
